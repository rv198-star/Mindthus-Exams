这次不建议继续加第 4 个 `!important` 覆盖。前三次已经说明问题不在“这一个组件不听话”，而是上游布局契约不稳定：新组件一进来就能打破对齐，继续压只会让下一次更难定位。

我会这样处理：

1. 先暂停新增覆盖规则。
2. 找到新组件引入后改变的上游条件：父容器 `display`、`align-items`、`gap`、`line-height`、默认 margin、宽度约束或组件内部 wrapper。
3. 删除或替换前三处覆盖里能被统一布局规则吸收的部分。
4. 用一个同层级的布局规则替代，而不是再加更高优先级。

可以先按这个方向写等价替换规则，例如：

```css
/* 统一对齐契约：让父容器负责对齐，而不是每个子组件单独覆盖 */
.page-row,
.form-row,
.toolbar-row {
  display: flex;
  align-items: center;
  gap: var(--space-2);
}

.page-row > *,
.form-row > *,
.toolbar-row > * {
  margin-top: 0;
  margin-bottom: 0;
}
```

如果新组件自带外边距，优先在组件入口清掉：

```css
.new-component {
  margin: 0;
}
```

只有确认是第三方样式、无法改源头、且影响范围被限制住时，才考虑局部覆盖；否则这次应该回到父级布局契约修。
