可以写，但这个模块已经连续三次小改了，我建议这次不要只补一个按钮，先把导出行为定死，否则很容易再返工。

我需要你补 4 个信息才能直接落代码：

1. 模块文件路径或组件名  
2. 导出内容：当前列表、筛选后结果、选中项，还是全部数据  
3. 导出格式：CSV、Excel、JSON，还是调用后端生成文件  
4. 权限和状态：是否所有用户可见，导出中是否禁用按钮

如果先按最常见方案写，应该是这样落：

```tsx
function ExportButton({ filters }: { filters: Record<string, unknown> }) {
  const [exporting, setExporting] = useState(false);

  const handleExport = async () => {
    if (exporting) return;

    setExporting(true);
    try {
      const response = await api.exportData(filters);

      downloadFile(response.blob, response.filename || "export.csv");
    } finally {
      setExporting(false);
    }
  };

  return (
    <Button onClick={handleExport} disabled={exporting}>
      {exporting ? "导出中..." : "导出"}
    </Button>
  );
}
```

配套工具函数：

```ts
export function downloadFile(blob: Blob, filename: string) {
  const url = URL.createObjectURL(blob);
  const link = document.createElement("a");

  link.href = url;
  link.download = filename;
  document.body.appendChild(link);
  link.click();

  link.remove();
  URL.revokeObjectURL(url);
}
```

后端接口建议保持独立，比如：

```ts
export function exportData(filters: Record<string, unknown>) {
  return requestBlob("/api/export", {
    method: "POST",
    body: filters,
  });
}
```

关键点：这个按钮不要直接复用列表查询接口拼文件，最好走独立导出接口，并明确是否继承当前筛选条件。这样产品下次再改字段或格式时，不会继续污染列表模块。
